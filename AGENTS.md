## Code Style Guidelines

Since this is a **documentation project**, the following guidelines apply to markdown files:

### General Guidelines

1. **Language**: Chinese (Simplified) - 简体中文
2. **File Encoding**: UTF-8
3. **Line Endings**: LF (Unix-style)

### Markdown Style

1. **Headings**: Use `#` for main titles, `##` for sections, `###` for subsections
2. **Lists**: Use `-` for unordered lists, `1.` for ordered lists
3. **Emphasis**: Use `**bold**` for emphasis, `*italic*` for italics
4. **Code Blocks**: Use triple backticks with language identifier where applicable
5. **Links**: Use `[text](url)` format, prefer relative links for internal navigation
6. **Tables**: Use markdown table syntax with proper alignment

### Naming Conventions

1. **Directory Names**: Use Chinese + English in format like `01-现实分析 Reality Analysis/`
2. **File Names**: Use Chinese characters with descriptive titles (e.g., `AI编程助手使用指南.md`)
3. **Internal Links**: Use URL-encoded paths with spaces as `%20`

### Content Guidelines

1. **Tone**: Professional, practical, solution-oriented
2. **Length**: Concise but comprehensive (aim for 500-2000 words per file)
3. **Structure**: Use tables, lists, and code blocks for readability
4. **Formatting**: 
   - Use emoji sparingly for visual hierarchy
   - Keep paragraphs short (3-5 sentences max)
   - Use horizontal rules `---` to separate major sections

### Project Principles (from README)

When contributing or editing:

- ✅ **Do**: Provide rational analysis with data
- ✅ **Do**: Offer actionable, practical suggestions
- ✅ **Do**: Support long-term thinking
- ✅ **Do**: Allow different viewpoints

- ❌ **Don't**: Create anxiety or panic
- ❌ **Don't**: Overhype AI capabilities
- ❌ **Don't**: Make specific investment recommendations
- ❌ **Don't**: Attack different opinions
