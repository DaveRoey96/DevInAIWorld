# 如何高效使用 Copilot/Claude/GPT

> 工具用对了，效率翻倍。

## Copilot - 代码补全专家

### 核心使用场景

1. **实时补全**
   - 等待 AI 建议
   - 按 Tab 接受
   - 继续输入筛选

2. **Alt+\\ 触发建议**
   - 手动触发
   - 适合复杂场景

3. **聊天模式**
   - Ctrl+Shift+i
   - 选中代码后提问

---

### 高效技巧

**1. 让 Copilot 理解你的代码风格**

```python
# 在项目根目录创建 .github/copilot-instructions.md
# 告诉它你的代码规范
```

**2. 使用注释引导生成**

```javascript
// TODO: 实现用户认证逻辑，使用 JWT
// 注意：密码要用 bcrypt 加密
function authenticateUser(username, password) {
  // Copilot 会根据注释生成代码
}
```

**3. 批量生成**

```python
# 写一个测试文件的基本结构，Copilot 会帮你填充
class TestUserService:
    def test_create_user(self):
        pass
    
    def test_get_user(self):
        pass
```

---

### 适合的场景

- ✅ 样板代码
- ✅ 常见模式
- ✅ 重复性代码
- ✅ 单元测试
- ❌ 复杂业务逻辑
- ❌ 架构设计
- ❌ 不熟悉的领域

---

## Claude - 对话推理专家

### 核心使用场景

1. **复杂问题分析**
   - Bug 根因分析
   - 架构决策咨询
   - 代码审查

2. **长对话**
   - 深入讨论
   - 迭代优化
   - 思维梳理

3. **文档生成**
   - 技术文档
   - README
   - API 文档

---

### 高效技巧

**1. 给足上下文**

```
❌ 帮我优化这段代码
✅ 帮我优化这段 Python 代码（完整代码），这是一个用户认证模块，使用 FastAPI，目标是提高查询性能

```

**2. 指定输出格式**

```
用 Markdown 表格输出：
- 三个缓存策略
- 适用场景
- 优缺点
```

**3. 分步对话**

```
Step 1: 先问"Redis 缓存穿透是什么"
Step 2: 再问"如何预防"
Step 3: 最后问"Python 代码示例"
```

---

### 适合的场景

- ✅ Bug 分析
- ✅ 架构咨询
- ✅ 代码审查
- ✅ 技术文档
- ✅ 复杂逻辑实现
- ❌ 快速代码补全（太慢）

## 工具组合策略

### 日常开发工作流


## 常见错误

### ❌ 过度依赖

- 什么问题都问 AI
- 不自己思考
- 失去独立能力

### ❌ 不验证

- AI 说的就是对的
- 直接复制使用
- 不测试不审查

### ❌ 上下文不足

- 期望 AI 猜到你的需求
- 不提供足够信息
- 得到的答案不准确

---

## 最佳实践

1. **精通一个工具**：比会用多个更有效
2. **保持独立思考**：AI 是助手，不是大脑
3. **验证一切**：AI 也会错
4. **持续学习**：工具在进化，你也要进化
