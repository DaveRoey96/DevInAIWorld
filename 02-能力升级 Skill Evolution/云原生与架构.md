# 云原生与架构

> 云原生是现代软件的基础设施。

## 什么是云原生

### 核心理念

- 容器化
- 微服务
- 声明式 API
- 可观测性
- 韧性

---

## 核心技术栈

### 1. 容器化 - Docker

**必须掌握**：
- Dockerfile 编写
- 镜像构建
- 网络配置
- 存储管理

---

### 2. 编排 - Kubernetes

**核心概念**：
- Pod/Deployment/Service
- ConfigMap/Secret
- Ingress
- HPA/VPA

**进阶**：
- Helm
- Operators
- CRD

---

### 3. 云服务

**AWS**：
- EC2, ECS, EKS
- Lambda
- RDS, DynamoDB
- S3

**GCP**：
- GKE
- Cloud Run
- BigQuery

**Azure**：
- AKS
- Azure Functions

---

### 4. 基础设施即代码

**Terraform**：
- 声明式
- 可版本控制
- 可复用

** Pulumi/Ansible**：
- 基础设施自动化

---

## 架构模式

### 1. 微服务

**优点**：
- 独立部署
- 技术异构
- 独立扩展

**挑战**：
- 分布式系统复杂性
- 服务治理
- 数据一致性

---

### 2. 服务网格

**Istio**：
- 流量管理
- 安全
- 可观测性

---

### 3. 事件驱动

**消息队列**：
- Kafka
- RabbitMQ
- Pulsar

---

### 4. Serverless

**场景**：
- 事件触发
- 低流量应用
- 快速原型

---
