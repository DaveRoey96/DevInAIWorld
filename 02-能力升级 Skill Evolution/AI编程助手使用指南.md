> AI不是替代你，而是放大你。

## 主流AI编程工具

### GitHub Copilot

**特点**：
- 代码补全为主
- 与 IDE 深度集成
- 支持多种语言

**适用场景**：
- 日常编码补全
- 样板代码生成
- 快速实现常见模式

**费用**：
- 个人：$10/月
- 学生：免费
- 企业：$19/月

---

**特点**：
- 对话式交互

**适用场景**：
- 代码审查
- 架构咨询
- Bug 分析
- 文档生成

**费用**：

## 核心使用原则

### 1. 把它当助手，不是替代品

- AI 生成代码 → 你审核 → 你决定是否使用
- 不要盲目复制 AI 的答案
- 始终保持判断力

### 2. 提供足够的上下文

- 描述清楚你想要什么
- 给出相关代码片段
- 说明技术栈和环境

### 3. 迭代优化

- 第一次回答可能不完美
- 继续追问、补充信息
- 引导 AI 往正确方向走

### 4. 验证和测试

- AI 也会出错
- 生成的代码一定要测试
- 重要逻辑要自己理解

## 效率提升技巧

### Prompt 技巧

**❌ 不好**：
```
写一个登录功能
```

**✅ 好**：
```
用 Python + FastAPI 写一个登录接口：
1. 接受 username/password
2. 用 bcrypt 加密密码
3. 返回 JWT token
4. 包含单元测试
```

---

### 分步操作

**复杂任务**：
```
1. 先让 AI 生成骨架
2. 再逐步填充细节
3. 最后让它帮你审查
```

---

### 利用上下文

- 在 Cursor 中，可以用 @ 引用整个代码库
- 在 Claude/Copilot 中，可以粘贴相关代码
- 上下文越多，答案越准确

## 常见使用场景

### 1. 代码生成

**Prompt**：
```
用 React + TypeScript 写一个用户列表组件：
- 表格展示用户信息
- 支持分页
- 支持搜索
- 使用 Ant Design
```

---

### 2. Bug 修复

**Prompt**：
```
帮我分析这个错误：
```
Error: Cannot read property 'map' of undefined
```
相关代码：
```javascript
const users = data.users;
return users.map(u => <UserCard user={u} />);
```
可能是什么原因？如何修复？
```

---

### 3. 代码审查

**Prompt**：
```
帮我审查这段代码：
```python
def process_payment(user_id, amount):
    user = get_user(user_id)
    if user.balance >= amount:
        user.balance -= amount
        save_user(user)
        return True
    return False
```
有什么问题？安全、性能、并发方面？
```

---

### 4. 技术学习

**Prompt**：
```
我想学习 Redis 缓存：
- 什么场景适合用缓存
- 缓存穿透/击穿/雪崩是什么
- Python 中如何优雅地使用 Redis
```

## 注意事项

### ⚠️ 安全问题

- 不要把敏感信息发给 AI
- 涉及密钥、密码的代码要小心
- 内部代码可以用本地部署的 AI

### ⚠️ 版权问题

- Copilot 生成的代码可能有版权争议
- 重要商业项目注意检查
- 开源项目相对安全

### ⚠️ 依赖问题

- AI 生成的代码可能引入不必要的依赖
- 注意检查依赖的安全性
- 评估是否真的需要

---

## 最佳实践

1. **选择主力工具**：精通一个工具比会用多个好
2. **建立自己的 Prompt 库**：常用场景保存模板
3. **持续学习**：AI 工具更新很快
4. **保持批判**：AI 不是完美的
