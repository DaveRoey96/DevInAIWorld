# 技术方案设计

> 方案决定成败。

## 什么是技术方案

### 定义

技术方案 = 把业务需求转化为可执行的技术计划

### 内容

1. **背景**
   - 业务背景
   - 问题描述
   - 目标

2. **方案**
   - 技术选型
   - 架构设计
   - API 设计
   - 数据模型

3. **实现**
   - 开发计划
   - 里程碑
   - 风险点

---

## 方案设计原则

### 1. 合适 > 完美

```
业务初期：简单方案
业务发展：可扩展方案
业务成熟：精细化方案
```

---

### 2. 简单 > 复杂

```
能简单实现就不复杂化
过度设计是最大的浪费
```

---

### 3. 渐进 > 一步到位

```
先 MVP
再迭代
```

---

### 4. 权衡 > 理想

```
没有最好，只有最适合
考虑约束：时间、成本、人员
```

---

## 方案评审

### 评审关注点

1. **完整性**
   - 覆盖所有场景
   - 考虑边界情况

2. **可行性**
   - 技术上能实现
   - 资源足够

3. **风险**
   - 识别风险
   - 有应对措施

4. **可维护性**
   - 易于理解
   - 易于修改

## AI 辅助方案设计

### AI 能帮

- 提供技术选型建议
- 给出参考模式
- 生成方案草稿

### AI 不能帮

- 理解你的具体业务
- 了解团队约束
- 判断优先级

---

## 实践建议

### 1. 多看优秀方案

- 开源项目设计文档
- 公司内部分享
- 技术博客

### 2. 参与方案评审

- 学习他人思路
- 培养批判思维
- 积累经验

### 3. 写方案

- 从小方案开始
- 不断迭代
- 总结复盘
